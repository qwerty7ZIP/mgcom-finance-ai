# MGCOM Finance AI — Документация

## Обзор проекта

**MGCOM Finance AI** — внутренний веб‑сервис для управленческой команды, который позволяет:

- **Задавать вопросы на русском языке** об операционной и коммерческой деятельности компании.
- **Получать интерактивные таблицы** (клиенты, контакты, тендеры и др.) с возможностью фильтрации, сортировки, пагинации и экспорта.
- **Уточнять выборки через ИИ‑чат** или напрямую через UI‑фильтры в таблице.

Текущая версия реализует **MVP‑прототип** на базе **БД Supabase (PostgreSQL)** и интегрируется с **YandexGPT** для интерпретации запросов пользователя в структуру запроса к данным.

## Структура документации

- `PLAN.md` — общий продуктовый и технический план развития MGCOM Finance AI (цели, MVP, дорожная карта).

- `DATA_MODEL.md` — описание источников данных, логических таблиц и ожидаемых связей.
- `API_AND_INTEGRATIONS.md` — описание публичных API‑эндпоинтов и интеграции с YandexGPT.

## Быстрый старт для разработчика

1. Прочитать `PLAN.md` для понимания целей и MVP.
2. Ознакомиться с `ARCHITECTURE.md` и `DESIGN_SYSTEM.md`, чтобы понимать границы системы и визуальные принципы.
3. Следовать инструкциям из `SETUP_AND_RUN.md` для локального поднятия проекта.
4. Использовать `API_AND_INTEGRATIONS.md` при работе с ИИ‑слоем и таблицами данных.

## Текущее состояние

- Реализован прототип фронтенда на **Next.js + TypeScript** с:
  - основным дашбордом (`Dashboard`),
  - табличным компонентом (`DataTable`) с фильтрами, сортировкой, пагинацией и экспортом в CSV,
  - боковой панелью чата (`ChatPanel`), связывающейся с `/api/chat`,
  - переключателем темы (`ThemeToggle`) и корпоративным UI.
- Есть серверные роуты:
  - `/api/data` — получает данные из БД Supabase по логической таблице (`clients`, `contacts`, `tenders`) и возвращает их в формате `{ columns, rows }`;
  - `/api/chat` — общается с **YandexGPT**, возвращает текст ответа и JSON‑структуру запроса к данным (`TableRequest`), на основе которой настраивается таблица.
- Все рабочие данные хранятся в **Supabase**; импорт из Excel‑файлов рассматривается как отдельный офлайн‑процесс и не участвует в рантайме приложения.
