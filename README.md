# Документация MGCOM Finance AI

Этот каталог содержит полную документацию по проекту **MGCOM Finance AI** — внутреннему сервису для работы с корпоративными данными (клиенты, контакты, тендеры) и ИИ‑чатом на основе Supabase PostgreSQL и YandexGPT.

## Структура документации

- `README.md` — этот файл, обзор документации и навигация.
- `PLAN.md` — продуктовый и технический план развития:
  - цели проекта;
  - ключевые сценарии (таблицы, чат, аналитика);
  - этапы внедрения и дальнейшие улучшения.
- `ARCHITECTURE.md` — архитектура системы:
  - общая схема фронтенда и бэкенда;
  - взаимодействие Next.js, Supabase и YandexGPT;
  - описание основных модулей (`Dashboard`, `DataTable`, `ChatPanel`, API‑роуты);
  - паттерны и принятые решения.
- `DATA_MODEL.md` — модель данных:
  - описание таблиц **clients**, **contacts**, **tenders** в Supabase;
  - связи между ними;
  - правила трактовки полей (например, какие поля считаются датами/суммами).
- `API_AND_INTEGRATIONS.md` — API и интеграции:
  - `/api/data` — доступ к данным из Supabase (GET/POST, фильтрация, сортировка, limit);
  - `/api/chat` — интеграция с YandexGPT (формат `tableRequest`, нормализация дат и limit);
  - используемые внешние сервисы.
- `FRONTEND_UI.md` — фронтенд и UI‑паттерны:
  - структура страниц и компонентов;
  - поведение таблиц (фильтры, пагинация, скрытие колонок, экспорт CSV);
  - ИИ‑чат, подсказки, режимы отображения.
- `DESIGN_SYSTEM.md` — визуальный стиль:
  - цветовая схема, шрифты, принципы светлой/тёмной темы;
  - правила для форм, таблиц, кнопок, состояний загрузки;
  - ссылки на `ui-ux-pro-max` и использованные решения.
- `SETUP_AND_RUN.md` — запуск и деплой:
  - требования к окружению;
  - настройка переменных среды (`.env.local`);
  - запуск локально (`npm run dev`) и сборка/старт в проде (`npm run build`, `npm run start`);
  - рекомендации по переносу на хостинг/домен.
- При необходимости:
  - дополнительные файлы (например, `SECURITY.md`, `MIGRATIONS.md`, `ANALYTICS.md`) можно добавлять сюда же с описанием своих задач.

## Рекомендуемый порядок чтения

1. `PLAN.md` — чтобы понять **зачем** и **что** делает система.
2. `ARCHITECTURE.md` — чтобы увидеть общую картину по слоям (фронтенд, API, БД, ИИ).
3. `DATA_MODEL.md` — чтобы разобраться с таблицами Supabase и полями.
4. `FRONTEND_UI.md` + `DESIGN_SYSTEM.md` — для разработки и доработки интерфейса.
5. `API_AND_INTEGRATIONS.md` — при работе с API и интеграциями (YandexGPT, Supabase).
6. `SETUP_AND_RUN.md` — при настройке окружения, локального запуска и деплоя.

## Краткое описание проекта

**MGCOM Finance AI** — это:

- единая точка доступа к данным клиентов, контактов и тендеров (Supabase PostgreSQL);
- умные таблицы с фильтрами, сортировкой, пагинацией, скрытием системных колонок и экспортом в CSV;
- ИИ‑чат на базе YandexGPT, который:
  - понимает запросы на русском языке;
  - строит структурированные запросы `tableRequest` (таблица, фильтры, сортировка, limit);
  - умеет работать с относительными периодами («прошлый месяц», «последние 3 месяца» и т.п.).

Документация в этой папке должна оставаться **актуальной** по мере развития проекта: при изменении архитектуры, схемы данных, UI или интеграций — сначала вносим изменения в код, затем обновляем соответствующие `.md`‑файлы тут.
